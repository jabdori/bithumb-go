# 웹소켓 에러

## 개요

빗썸 WebSocket API에서 발생할 수 있는 에러 형식과 에러 코드에 대한 설명입니다.

## Format

에러 발생 시 JSON 형식의 에러 메시지가 내려갑니다.

## Response

에러 응답은 다음과 같은 구조로 전달됩니다:

| 필드명 | 내용 | 타입 |
|--------|------|------|
| error | 에러 객체 | `Object` |
| error.name | 종류 | `String` |
| error.message | 메세지 | `String` |

## 에러 종류

### 1. WRONG_FORMAT

형식이 올바르지 않을 때 발생합니다.

```json
{
  "error": {
    "name": "WRONG_FORMAT",
    "message": "Format 이 맞지 않습니다."
  }
}
```

### 2. NO_TICKET

티켓이 존재하지 않거나 유효하지 않을 때 발생합니다.

```json
{
  "error": {
    "name": "NO_TICKET",
    "message": "티켓이 존재하지 않거나, 유효하지 않습니다."
  }
}
```

### 3. NO_TYPE

`type` 필드가 존재하지 않을 때 발생합니다.

```json
{
  "error": {
    "name": "NO_TYPE",
    "message": "type 필드가 존재하지 않습니다."
  }
}
```

### 4. NO_CODES

`codes` 필드가 존재하지 않을 때 발생합니다.

```json
{
  "error": {
    "name": "NO_CODES",
    "message": "codes 필드가 존재하지 않습니다."
  }
}
```

### 5. INVALID_PARAM

`codes` 필드가 비어 있을 때 발생합니다.

```json
{
  "error": {
    "name": "INVALID_PARAM",
    "message": "codes 필드가 비어 있습니다."
  }
}
```

### 6. INVALID_PARAM_FORMAT

지원하지 않는 포맷일 때 발생합니다.

```json
{
  "error": {
    "name": "INVALID_PARAM",
    "message": "{}은 지원하지 않는 포맷입니다."
  }
}
```

## 에러 처리 권장사항

1. **항상 에러 객체 확인**: WebSocket 메시지 수신 시 `error` 필드가 있는지 항상 확인하세요.
2. **에러 타입별 분기 처리**: `error.name` 필드를 사용하여 각 에러 타입에 맞는 적절한 처리를 구현하세요.
3. **사용자 피드백**: `error.message`를 사용하여 사용자에게 명확한 에러 정보를 제공하세요.
4. **재연결 로직**: 일시적인 연결 문제인 경우 적절한 대기 시간 후 재연결을 시도하세요.
